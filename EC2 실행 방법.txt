디버그용 실행 방법 ( -w 4 부분은 워커 4개 쓴다는 뜻, 제거 가능 )
gunicorn -w 4 -k uvicorn.workers.UvicornWorker AI.controller:app

일반 실행 방법 ( 백그라운드 실행 ) ( -w 4 부분은 워커 4개 쓴다는 뜻, 제거 가능 )
nohup gunicorn -w 4 -k uvicorn.workers.UvicornWorker AI.controller:app &

워커를 4개 쓰면 요청이 1개일 때는 오히려 성능 저하

pm2의 --watch와 같은 기능 -> --reload
nohup gunicorn -w 2 -k uvicorn.workers.UvicornWorker --reload AI.controller:app &


pm2 설치하고 pm2.json 실행하는거로 바꿈 - 로그를 터미널에서 바로 볼 수 있음
주의점 : pm2 명령어에 --watch 붙이면 에러남. json 안에 -reload 넣어놓았으니까 그냥 pm2 start pm2.json 만 할 것!
